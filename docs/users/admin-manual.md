# Admin Manual

This document summarizes the main admin-facing responsibilities in the platform.

## Core Tasks

- review marketplace health and analytics
- moderate listings and platform activity
- manage disputes and support workflows
- inspect operational or system-level surfaces
- review fraud, trust, and audit-related signals

## Typical Journey

1. Open the admin area.
2. Review metrics, alerts, and outstanding issues.
3. Inspect operational or moderation queues.
4. Take corrective or support actions.
5. Record or follow up on issues that need engineering or policy changes.

## Notes

- admin flows are broader on web than mobile
- operational trust depends on working navigation, accurate data contracts, and clear status visibility
